A bit of history

  • chroot: security
  • *BSD Jails: security
  • Solaris (light) zones: security and resource management
  • Virtualization: resource management
  • Linux Containers (lxc): security and resource management

Why lxc ?

  • Low cost isolation: iPhone and Android app splitted by uid
  • Process babysitting: systemd, Google Chrome
  • Customized isolation level (cpu, memory, io, devices): Google Compute Engine
But
  • All containers share the same linux kernel (as in Solaris light zones)

Why should you care ?

  • You already know Linux ? You almost know lxc
  • It is supported by all major distributions
  • Seamless: all applications work out of the box
  • Monitoring !!!
  • No commercial licensing issue
  • Low overhead !

But not designed for

  • OS virtualization
  • HW clustering (not yet at least)
  • Making coffee (you already have Java for that)
